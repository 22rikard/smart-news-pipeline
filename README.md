# Smart-News-Pipeline 🎬

**基於多模態大模型驅動的全自動新聞生產管線**

</div>

## 🚀 解決需求

在傳統新聞內容生產中，素材採集、事實提煉、劇本撰寫以及音畫同步是極度耗時的重複勞動。現有的自動化工具大多缺乏對影像內容的深度語義理解，無法有效從散亂的原始影片中萃取新聞價值。

**Smart-News-Pipeline** 秉持 **Video-First（影片優先）** 與 **Human-in-the-Loop（人機協作）** 的核心理念，實現從「多源原始影片」到「高品質新聞成片」的智能化生產轉型，將 AI 轉化為新聞編輯的超級助理。

---

## 💡 設計理念

*   **Video-First（影片生新聞）**：跳過「先寫劇本再找畫面」的傳統邏輯。系統直接從多源原始素材中萃取事實與資訊，並根據影像內容自動重編新聞劇本，確保內容具備真實的影像依據。
*   **Human-in-the-Loop（人機協作）**：系統負責繁重的情報提取、採集與自動化編譯，但在「事實核查」、「劇本定調」與「最終審查」等核心節點，堅持由人類編輯介入決策，確保新聞的專業判斷與不可取代的靈魂。
*   **介面驅動與彈性補救**：模組之間完全透過標準 JSON 契約溝通，內建三級補救機制，確保在素材極度匱乏時仍能維持高品質的新聞敘事流暢度。

---

## 🌟 核心功能

1.  **影像語義與事實提煉**：自動掃描原始新聞素材，辨識鏡頭切換，並同步執行語音辨識與畫面解析，提煉出結構化的「事實特徵庫」。
2.  **新聞劇本智能重編**：根據提煉的事實摘要，大模型自動生成符合頻道調性的分鏡劇本與高擬真主播配音，實現從非結構化數據到專業敘事的轉化。
3.  **動態多級補救機制**：當本地素材無法支持新聞事實時，系統自動生成精準提示詞並調用 Text-to-Video API 生產視覺補強，拒絕生硬剪輯。
4.  **毫秒級音畫同步**：採用語義映射算法將正確劇本與配音音軌進行精準對齊，確立完美的剪輯基標。
5.  **無感化自動渲染**：底層集成 FFmpeg 執行無損切割、格式統一與高畫質 ASS 樣式字幕硬燒錄，實現一鍵成片。

---

## 🛡️ 技術亮點：三級補救機制 (Fallback Mechanism)

為確保影片敘事與旁白在數學層面上的絕對對齊（誤差 < 0.1 秒），系統實作了自動化的彈性路徑：

1.  **機制 A（精確截斷）**：真實素材長於敘事時，自動執行物理切割以匹配節奏。
2.  **機制 B（語義串聯）**：單一素材短於敘事時，自動檢索同主題片段進行連貫拼接。
3.  **機制 C（AIGC 終極補強）**：**當視覺證據缺失時**，系統自動輸出 `AIGC_REQUIRED` 任務，撰寫高質量英文提示詞，呼叫生成式 AI 憑空生成符合當下語境的新聞畫面。

---

## 🏗️ 實現架構 (5 步流水線)

1.  **事實萃取 (`extractor.py`)**：利用多模態模型，將原始像素轉化為包含畫面描述與主題標籤的 JSON 知識庫。
2.  **時軸同步 (`sync.py`)**：負責配音與劇本的毫秒級校準，產出 `final_timeline.json`。
3.  **智能導演 (`director.py`)**：執行語義匹配邏輯與補救機制，輸出核心決策清單。
4.  **輔助生成 (`core_generator.py`)**：處理 AIGC 生成任務，調用外部 API 補齊畫面缺口並更新路徑。
5.  **高效渲染 (`render.py`)**：讀取決策清單，執行物理切割、拼接與字幕硬燒錄。
