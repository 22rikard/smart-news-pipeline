# Smart-News-Pipeline 🎬

**基於大模型驅動的全自動新聞影片剪輯與生成管線**

</div>

## 🚀 解決需求

在傳統的新聞短影片或資訊類內容生產過程中，尋找素材、畫面粗剪、精細的音畫同步以及字幕製作是極度耗時的體力勞動。現有的自動化剪輯工具大多缺乏對內容的深度語義理解，導致新聞剪輯效果生硬且不準確。

**Smart-News-Pipeline** 提供了一套完整的解決方案，實現從「原始素材 + 純文字腳本」到「廣播級高品質新聞成片」的全自動化生產流程。系統能真正「理解」新聞畫面與旁白，大幅降低內容創作的門檻與時間成本。

---

## 💡 設計理念

*   **流水線與關注點分離 (Pipeline & Separation of Concerns)**：將極度複雜的新聞影片生產流程拆解為 5 個相互獨立的引擎。每個引擎僅負責單一職責，這不僅降低了系統的工程複雜度，更有效緩解了大語言模型 (LLM) 在處理超長上下文時容易產生的「幻覺」問題。
*   **介面驅動開發 (Interface-Driven)**：模組之間完全透過標準化的 JSON 數據契約進行通訊。這種設計使得系統具備極強的解耦性與擴展性，未來可以輕鬆替換任何一個環節的底層算法（例如更換更強的新聞影片生成 API）。
*   **彈性防呆與補救 (Resilience & Fallback)**：系統拒絕「為剪而剪」。當現有素材庫無法滿足劇本的語義或時長需求時，系統不會強行拼接無關畫面，而是會觸發自動化的生成式補償路徑，確保最終產出的視覺合理性與高品質新聞內容。

---

## 🌟 核心功能

1.  **新聞語義索引化**：自動掃描原始素材，精準辨識鏡頭切換（Hard cut），並為每個片段生成詳細的視覺描述與語義標籤。
2.  **高精度音畫對齊**：採用雙步映射算法，將純淨腳本與音訊時間戳進行語義對齊，徹底解決傳統 ASR 字幕錯字多、斷句不準的痛點。
3.  **智能導演決策**：根據新聞旁白含義，自動從視覺知識庫中檢索最相關的鏡頭，並進行精確到毫秒的時間軸對齊。
4.  **動態多級補救機制**：在素材長度不符或完全短缺時，自動進行智能裁切、語義拼接，甚至即時調用 AIGC 引擎無中生有生產補充新聞畫面。
5.  **無感渲染引擎**：底層集成 FFmpeg 進行無損切割與拼接，支持外掛 `config.ini` 控制的高畫質 ASS 樣式字幕硬燒錄。

## 🛡️ 核心亮點：多級補救機制 (Fallback Mechanism)

這是本系統確保新聞影片質量與自動化穩定性的核心。在進行畫面匹配時，系統被賦予了一項極為嚴格的數學約束：**挑選出的新聞片段，其播放總時長必須精確等於旁白段落的時長（誤差 < 0.1 秒）。**

為了在現實素材匱乏的情況下滿足這項嚴苛條件，系統實作了智能的三級補救機制：

1.  **機制 A（素材過長 - 精確截斷）**：
    當選中最適配的真實素材長於該段旁白時，系統會自動計算時間差，並修改素材的 `clip_endTime`，將其精確截斷至符合新聞旁白的播放長度。
2.  **機制 B（素材過短 - 語義串聯）**：
    當選中的單一素材短於旁白時，系統不會讓畫面定格或黑屏。它會繼續從素材庫中尋找同主題、語義連貫的其他片段，將它們串聯放置在同一個渲染陣列中，直到播放總時長達標。
3.  **機制 C（終極補救 - AIGC 動態生成）**：
    如果素材庫極度匱乏，**完全找不到**與新聞語境相關的鏡頭，系統絕對不會勉強重複播放舊畫面或拿不相關的鏡頭湊數。
    *   **觸發生成任務**：系統會主動放棄本地檢索，輸出一個帶有 `AIGC_REQUIRED` 標記的特殊任務。
    *   **動態 Prompt 撰寫**：大模型會根據當下這句旁白的具體語境與情感，自動撰寫一段高質量的英文影片生成提示詞（Generation Prompt）。
    *   **非同步生成與無縫替換**：後續的 `core_generator.py` 引擎會攔截這個任務，自動調用（如 Runway, Luma 等）Text-to-Video API 憑空生成全新新聞畫面，下載後無縫替換掉決策表中的佔位符。這賦予了系統極強的「抗脆弱性」。

## 🏗️ 實現架構 (5 步流水線)

系統由以下五個核心模組構成，資料單向流動：

### 1. 視覺提取引擎 (`extractor.py`)
利用多模態大模型對原始新聞影片進行掃描。將無意義的像素轉化為機器可讀的視覺知識庫。
*   **介面產出**：包含精確時間戳、畫面描述 (`clip_frame`)、主題 (`topic`) 的結構化 JSON。

### 2. 時軸同步引擎 (`sync.py`)
負責將新聞配音與人類撰寫的 Markdown 腳本進行時間軸校準。
*   **介面產出**：帶有精確時間點位與 100% 正確文本的 `final_timeline.json`。

### 3. 智能決策中樞 (`director.py`)
系統的「大腦」，負責執行新聞語義匹配邏輯與上述的多級補救機制。
*   **介面產出**：包含具體影片切割時間與 AIGC 佔位符的混合決策表 `mapping.json`。

### 4. 輔助素材生成引擎 (`core_generator.py`)
專門處理導演中樞留下的 AIGC 生成任務。
*   **機制**：解析提示詞，調用外部影片生成 API，將產出的 `.mp4` 素材保存，並自動將 `mapping.json` 中的佔位符更新為真實路徑。

### 5. 高效渲染引擎 (`render.py`)
純粹基於 FFmpeg 的執行層，完全不含決策邏輯。
*   **機制**：讀取最終的 `mapping.json`，執行新聞影片切割、格式標準化（1080p/30fps）、音畫合成，並根據外掛設定檔渲染高畫質字幕。

## 🔌 數據契約範例

本系統所有的模組解耦，皆依賴於這份核心的剪輯決策列表 (`mapping.json`)。它展示了真實素材與 AIGC 生成任務是如何完美混合排布的：

```json
[
  {
    "narrative_startTime": "00:22.500",
    "narrative_endTime": "00:27.800",
    "text": "新聞旁白內容...",
    "narration_duration": 5.3,
    "video_clips": [
      {
        "source_file": "existing_broll.webm",
        "clip_startTime": "00:59.700",
        "clip_endTime": "01:04.000",
        "play_duration": 4.3
      },
      {
        "source_file": "AIGC_REQUIRED—generated_file.mp4",
        "generation_prompt": "Hyper-realistic 3D animation of a futuristic AI core...",
        "clip_startTime": "00:00.000",
        "clip_endTime": "00:01.000",
        "play_duration": 1.0
      }
    ]
  }
]
```
